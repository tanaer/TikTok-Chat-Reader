# 后台用户会员与配额人工调整

## 功能范围

管理后台“用户管理 → 详情”现在支持两类人工干预：

- 会员计划调整：管理员可直接给用户切换套餐，并设置新的到期时间。
- 配额覆盖调整：管理员可继续对“可建房间数 / 每日可添加次数”做永久或临时覆盖。

这样后台可以同时处理“会员权益”和“特殊个案配额”两种不同场景，避免只改配额却看不出用户真实会员状态。

## 会员计划调整规则

为了兼容现有配额系统与订阅历史，后台没有直接篡改所有历史记录，而是遵循下面的规则：

- 同套餐调整到期时间：直接更新当前生效中的那条订阅记录的 `end_date`。
- 切换到其他套餐：先把当前 `active` 订阅置为 `cancelled`，再新建一条新的订阅记录。
- 若填写的到期时间早于当前时间：新记录或当前记录会被标记为 `expired`，该用户会立即失去会员配额。
- 为避免脏数据影响配额计算，保存时会清理同一用户其它重复的 `active` 订阅记录。

这个策略的目标是保证“任意时刻最多只有一条真正生效的主套餐订阅”，从而和 `quotaService` 当前的“取最新一条 active 且未过期订阅”逻辑完全一致。

## 与配额系统的关系

当前用户总配额来源依旧是：

- 主套餐房间额度
- 生效中的扩容包额度
- 系统默认房间额度
- 管理员配额覆盖值（永久 / 临时）

后台手工调整会员后，会立即重新计算该用户的：

- 可建房间总额度
- 剩余额度
- 每日可添加上限
- 当前权益来源说明

因此，会员计划调整和配额调整可以叠加使用：

- 会员决定“基础权益”
- 配额覆盖决定“人工特批结果”

## 后台交互说明

用户详情弹窗中新增“会员”区域，包含：

- 当前生效会员摘要
- 套餐下拉选择
- 到期时间输入
- 保存会员调整按钮
- 订阅记录列表

保存提示文案会明确区分：

- 只是更新当前套餐到期时间
- 还是发生了套餐切换
- 是否已经被设置为立即过期

这样管理员在操作时能直观看到“当前权益”和“历史记录”是否一致。
