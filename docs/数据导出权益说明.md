# 数据导出权益说明

## 背景

用户分析页的导出能力属于套餐权益，不应只依赖前端按钮显示控制。

此前系统存在两处不一致：

- 前端仅识别 `feature_flags.export`
- 后端导出接口自行查询订阅，仅识别 `feature_flags.export`

这会导致后台套餐里如果沿用了历史键名 `data_export`，用户已开通权益但仍看不到导出按钮，或接口判断与前端展示不一致。

## 当前机制

从 2026-03-08 起，数据导出统一按以下规则判断：

1. 管理员始终允许导出。
2. 普通用户通过统一权益校验函数 `ensureUserPlanFeature()` 判断。
3. 数据导出同时兼容两个键名：`export` 与 `data_export`。
4. 用户分析页只在用户拥有该权益时展示导出按钮。
5. 后端导出接口 `GET /api/analysis/users/export` 也使用同一套权益校验，避免只隐藏按钮但接口仍可被绕过或误拦截。

## 影响范围

- 用户分析页面导出按钮显示
- `GET /api/analysis/users/export` 权限控制
- 套餐配置中的 `feature_flags.export` / `feature_flags.data_export`

## 安全与响应面

- 本次改动不新增敏感返回字段。
- 用户侧只获得是否能看到导出按钮的结果，不暴露后台套餐内部实现细节。
- 真正的权限控制仍以后端接口校验为准。
