# AI客户分析异步任务与点数方案补充

日期：2026-03-09

## 补充目的

补充说明当前版本里两类 AI 用户分析能力的交互与计费边界：

- `AI性格分析`：继续保留同步分析体验
- `房间AI客户分析`：采用后台异步任务 + 消息通知

## 当前能力拆分

### 1. AI性格分析

入口：`用户分析 -> 用户详情`

接口：`POST /api/analysis/ai`

特点：

- 使用旧版性格分析 Prompt
- 基于用户历史弹幕语料生成
- 结果绑定 `当前登录用户账号 + 目标用户`
- 用户前台继续同步等待结果
- 后台仍会生成 `AI工作任务` 记录，便于统一留痕

### 2. 房间AI客户分析

入口：`房间详情 -> 历史排行榜`

接口：

- `GET /api/rooms/:id/customer-analysis/:userId`
- `POST /api/rooms/:id/customer-analysis`

特点：

- 分析基于 `当前房间` 的客户上下文
- 首次生成或强制重算进入后台异步任务
- 完成后通过站内消息通知提醒
- 结果绑定 `当前登录用户账号 + 当前房间 + 目标用户`
- 后台 `AI工作中心` 可统一查看全部记录

## 为什么房间客户分析更适合异步

房间客户分析已不是简单弹幕总结，而是：

- 先由系统构造结构化客户上下文
- 再叠加最近弹幕语料
- 再输出价值判断、风险判断、维护动作和主播话术

所以它更适合：

- 后台排队处理
- 统一追踪状态
- 完成后消息提醒
- 在 `AI工作中心` 留痕

## AI 工作中心归档规则

当前统一使用 `ai_work_job`：

- `session_recap`：AI直播复盘
- `customer_analysis + analysisScene=personality`：AI性格分析
- `customer_analysis + analysisScene=room_customer`：房间AI客户分析

这意味着：

- 所有 AI 分析后台都能看到
- 用户分析和房间客户分析都归在 `用户` 分类里
- 房间复盘仍归在 `房间` 分类里

## 当前点数方案

### 已落地口径

- `AI性格分析`：`1 点 / 次`
- `房间AI客户分析`：`3 点 / 次`
- `AI直播复盘`：`20 点 / 场`
- 命中当前账号缓存再次查看：`0 点`
- 管理员发起：`0 点`

### 定价推导

按现有点数包的 CNY 心智口径，单点大致可按 `¥0.4 ~ ¥0.5` 估算：

- `1 点` ≈ `¥0.4 ~ ¥0.5`
- `3 点` ≈ `¥1.2 ~ ¥1.5`
- `20 点` ≈ `¥8 ~ ¥10`

当前定价刻意保留层级：

- `AI性格分析` 保持轻量引流
- `房间AI客户分析` 明显高于性格分析，体现策略价值
- `AI直播复盘` 继续保持高客单价定位

## 偏贵策略建议

后续如果增加更强交付物，建议宁可略贵，不要把价格锚死在过低区间：

- 批量客户分析：`20 ~ 30 点 / 批`
- 高价值客户召回方案包：`30 ~ 50 点 / 批`
- 房间客户分层打法包：`50 点+ / 房`

原则：

- 只要产出能直接指导运营动作，就不按“便宜工具”定价
- 只要加入更多系统特征、更多策略和成品话术，就应同步提高点数
- 以营收优先为导向，优先建立高价值 AI 交付心智

## 用户态安全要求

即使进入异步任务，用户侧返回仍必须坚持白名单：

- 可以返回任务状态、进度、摘要、安全跳转地址
- 不返回原始 Prompt
- 不返回原始 `customerContextJson`
- 不返回原始 `requestPayloadJson` / `resultJson`
- 不返回内部调试字段
