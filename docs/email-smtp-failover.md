# 邮箱 SMTP 多服务与自动切换

## 变更目标

管理后台将原本挂在“系统设置”里的单一 SMTP 配置拆分为独立“邮箱服务”菜单，支持：

- 配置多个 SMTP 服务
- 指定默认服务
- 默认服务失败后自动切换到其他启用服务
- 对失败节点设置冷却时间，避免短时间内反复命中异常节点
- 单独管理“注册邮箱验证”开关

## 数据结构

新增 `smtp_services` 表，用于保存 SMTP 服务连接信息和运行状态：

- 基础连接信息：`host`、`port`、`secure`、`username`、`password`
- 发件信息：`from_email`、`from_name`
- 管理状态：`is_active`、`is_default`
- 运行状态：`last_status`、`last_error`、`cooldown_until`
- 统计字段：`call_count`、`success_count`、`fail_count`、`avg_latency_ms`

历史单 SMTP 配置仍保存在 `settings` 表中。系统首次读取新服务列表时，会自动将旧配置迁移为一个默认服务，避免升级后丢失邮件能力。

## 发送策略

邮件发送流程改为：

1. 读取所有启用的邮箱服务
2. 先尝试默认服务
3. 如果默认服务失败，则按可用性继续尝试其他启用服务
4. 失败服务会记录错误信息，并进入短暂冷却
5. 冷却中的节点会被自动降级到后续候选，避免持续阻塞用户请求

当前冷却时长默认为 5 分钟，可通过环境变量 `SMTP_FAILURE_COOLDOWN_MS` 调整。

## 管理面板

“邮箱服务”菜单包含两部分：

- 左侧服务列表：查看状态、错误、调用统计、设为默认、启用/禁用、测试连接、测试发信
- 右侧策略与编辑器：控制注册邮箱验证开关，并新增或编辑 SMTP 服务

“系统设置”页只保留基础设置和登录安全配置，不再直接管理 SMTP 连接信息。

## 安全边界

所有邮箱服务管理接口均在 `admin` 路由下，仅管理员可访问。

用户侧注册、找回密码、改绑邮箱仍只经过既有业务接口，不会暴露 SMTP 服务列表、密码、错误明细或内部切换策略。
