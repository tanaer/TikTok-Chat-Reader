# AI工作中心与异步AI任务机制

日期：2026-03-08

## 目标

统一承接系统内耗时较长的 AI 任务，避免用户在前台同步等待，同时把任务执行日志、进度、结果状态集中沉淀到可追踪的后台工作流里。

当前已接入两类任务：

- `session_recap`：AI直播复盘
- `customer_analysis`：AI客户分析

## 总体原则

- 用户侧请求只负责提交任务或读取安全缓存，不暴露内部 Prompt、原始上下文或调试数据。
- 实际数值、时间、排行、分层标签都由系统计算，AI 只负责解释、总结、策略和话术。
- 所有异步任务都写入 `ai_work_job` 与 `ai_work_job_log`，便于调试与追溯。
- 任务完成或失败后，统一通过站内通知回推给用户。
- 真实扣点、结果落库、任务完成状态应尽量在同一事务内提交，避免“已扣点但任务未完成”。
- 对可重试任务，要为单个 job 建立幂等落库锚点，避免 stale worker 或重复执行导致重复扣点。

## 整体流程

### 1. 用户发起

#### AI直播复盘

用户在监控中心提交 `生成AI直播复盘` 后：

- 服务端校验登录状态、房间权限、套餐权益、场次有效性和 AI 点数。
- 如果命中可复用缓存，直接返回缓存结果。
- 如果已有 `queued/processing` 任务，直接复用该任务。
- 否则创建 `session_recap` 类型任务并返回 `202 Accepted`。

#### AI客户分析

用户在 `用户分析 -> 用户详情 -> AI客户分析` 提交后：

- 服务端校验登录状态、用户权限和房间范围。
- 命中个人缓存时，直接返回缓存结果。
- 如果已有 `queued/processing` 的客户分析任务，直接复用。
- 否则创建 `customer_analysis` 类型任务并返回 `202 Accepted`。

### 2. 后台 worker 执行

Web 进程启动后会开启 AI 工作队列轮询器：

- 定时从 `ai_work_job` 领取 `queued` 任务。
- 默认允许最多 `3` 个 AI 任务并发执行，可通过环境变量 `AI_WORKER_MAX_CONCURRENCY` 调整；代码层当前保护上限为 `6`。
- 将任务标记为 `processing` 并更新当前步骤、进度。
- 按不同任务类型分发到独立处理流程。
- 成功后写入结果缓存并标记 `completed`。
- 失败后写入错误信息并标记 `failed`。

### 3. 任务完成通知

任务结束后，系统会创建站内通知：

- `AI直播复盘已完成 / 处理失败`
- `AI客户分析已完成 / 处理失败`

通知里的跳转地址是安全白名单地址：

- 直播复盘：直达对应房间与场次的 `AI直播复盘` TAB
- 客户分析：直达 `/monitor.html?section=userAnalysis` 并打开目标用户详情

## 数据表设计

### `ai_work_job`

用于记录 AI 异步任务主状态，核心字段包括：

- `user_id`
- `job_type`
- `room_id`
- `session_id`
- `title`
- `status`
- `current_step`
- `progress_percent`
- `point_cost`
- `charged_points`
- `force_regenerate`
- `request_payload_json`
- `result_json`
- `error_message`
- `notification_sent`
- `queued_at`
- `started_at`
- `finished_at`

说明：

- `charged_points` 用于记录该任务最终实际扣点。
- 任务进入 `completed` 时，代表结果与扣点结算已经成功提交，不应再被重新扣点执行。

### `ai_work_job_log`

用于记录每个任务的阶段性日志，核心字段包括：

- `job_id`
- `phase`
- `level`
- `message`
- `payload_json`
- `created_at`

### `user_ai_analysis`

用于保存会员维度的 AI 用户分析结果，当前新增：

- `source_job_id`

说明：

- 同一个 AI job 写入分析结果时会带上 `source_job_id`，作为幂等锚点。
- 这样即便 worker 重试，同一 job 也不会重复插入结果或重复扣点。

## 日志追踪范围

### AI直播复盘

当前会记录这些关键阶段：

- 任务入队
- worker 领取任务
- 场次 recap 数据加载完成
- 高频弹幕筛选开始 / 完成 / fallback
- 主提示词渲染完成
- AI 请求发起 / 成功 / 失败
- 主分析解析完成
- 结果落库
- 任务完成 / 失败

### AI客户分析

当前会记录这些关键阶段：

- 任务入队
- 用户上下文准备完成
- 结构化上下文构建完成
- AI 主分析开始 / 完成
- 结果落库与缓存写入
- 扣点结果
- 任务完成 / 失败

## 前台体验

### 监控中心

- 用户提交后会看到“后台处理中 / 排队中”状态，而不是同步等待。
- 客户分析完成后可直接在用户详情里看到结构化分析结果。
- 直播复盘完成后可直接在房间详情里查看复盘内容。

### AI工作中心

前台 AI工作中心当前会统一展示两类任务：

- 用户：AI客户分析
- 房间：AI直播复盘

并支持：

- 状态筛选
- 分类筛选
- 自动轮询刷新
- 按任务安全跳转回监控中心对应页面

## 管理后台能力

后台 `AI工作中心` 用于集中查看：

- 所有 AI 工作任务
- 分类（用户 / 房间）
- 当前步骤与进度
- 请求参数
- 结果 JSON
- 全流程处理日志

当前后台已经支持：

- 按状态筛选
- 按分类筛选
- 按用户 / 房间 / 目标对象 / 标题搜索
- 查看任务详情与日志

## 安全边界

- 用户态接口必须使用白名单序列化，不直接返回原始数据库行。
- `request_payload_json`、`result_json`、Prompt、调试上下文仅在后台管理视图可见。
- 用户态任务列表只返回任务卡片所需字段，如状态、步骤、进度、价格、摘要、安全跳转地址。
- 不向普通用户暴露原始 `customerContextJson`、原始 AI 请求参数、原始模型返回体。
