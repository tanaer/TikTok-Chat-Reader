# AI工作中心与异步复盘机制

## 目标

为了解决 `生成AI直播复盘` 同步请求容易超时的问题，系统将直播复盘改造成后台异步任务，并提供统一的任务查看、日志追踪、通知提醒与提示词管理能力。

这次改造重点解决三件事：

- 用户点击生成后，接口立即返回，不再同步等待 AI 完成。
- 每个复盘任务的全流程都会写入后台管理面板，方便监听、调试和优化。
- 前台不再单独展示 `AI工作中心` 入口；AI直播复盘完成后会通过消息通知提醒，点击通知可直达对应房间的 `AI直播复盘` TAB。

## 整体流程

### 1. 用户提交任务

用户在监控中心的单场复盘区域点击 `生成AI直播复盘` 后：

- 服务端先校验登录状态、房间权限、套餐权益、场次有效性和 AI 点数。
- 如果当前场次已存在缓存复盘，直接返回缓存结果。
- 如果当前场次已经有 `queued/processing` 中的任务，直接复用已有任务，避免重复提交。
- 否则创建一条新的 `ai_work_job` 记录，并写入首条 `ai_work_job_log` 日志。

接口会返回 `202 Accepted`，前端立即提示用户任务已经启动。

### 2. 后台 worker 处理任务

Web 进程启动后会开启 AI 工作队列轮询器：

- 定时从 `ai_work_job` 表中领取 `queued` 任务。
- 标记任务进入 `processing`。
- 加载单场 recap 数据。
- 先对高频弹幕 Top50 做一次 AI 价值筛选。
- 再走主分析提示词，生成结构化 AI直播复盘。
- 保存 `session_ai_review` 缓存，并在需要时扣除 AI 点数。
- 将任务标记为 `completed` 或 `failed`。

## 数据表设计

### `ai_work_job`

用于记录 AI 异步任务主状态，核心字段包括：

- `user_id`
- `job_type`
- `room_id`
- `session_id`
- `title`
- `status`
- `current_step`
- `progress_percent`
- `point_cost`
- `charged_points`
- `force_regenerate`
- `request_payload_json`
- `result_json`
- `error_message`
- `notification_sent`
- `queued_at`
- `started_at`
- `finished_at`

### `ai_work_job_log`

用于记录每个任务的阶段性日志，核心字段包括：

- `job_id`
- `phase`
- `level`
- `message`
- `payload_json`
- `created_at`

## 日志追踪范围

当前 AI直播复盘任务会记录这些关键阶段：

- 任务入队
- worker 领取任务
- 场次 recap 数据加载完成
- 高频弹幕筛选开始 / 完成 / fallback
- 主提示词渲染完成
- AI 请求发起 / 成功 / 失败
- 主分析解析完成
- fallback 原因
- 结果落库
- 任务完成 / 失败

这些日志会展示在后台管理面板的 `AI工作中心` 中，方便排查：

- 哪一步最慢
- 哪一步最容易失败
- 哪个模型失败率高
- prompt 长度、输入规模、fallback 情况

## 前台体验改造

### 消息通知直达

前台不再额外展示 `AI工作中心` 菜单。

用户在提交 AI直播复盘 后，可以直接离开当前页面；任务完成或失败后，系统会发一条站内消息通知。点击通知会直接跳转到对应房间的 `AI直播复盘` TAB。

### 监控中心提示

当用户提交 AI直播复盘后：

- 当前复盘区域会切换成“后台处理中”状态。
- 会展示当前步骤、进度、说明文案。
- 明确提示用户无需等待，完成后会通过消息通知提醒。
- 已生成的复盘内容支持导出为图片或 PDF，方便转发给老板或团队留档。

## 通知机制

任务完成或失败后，系统会创建一条站内通知：

- 完成：通知用户复盘已可查看。
- 失败：通知用户查看失败原因并重新提交。

通知会写入安全的站内跳转地址，用户点击后可直接进入 `/monitor.html` 对应房间与场次，并自动切到 `AI直播复盘` TAB。

## 管理后台能力

后台管理面板新增 `AI工作中心` 区域，用于查看：

- 任务列表
- 当前步骤与进度
- 请求参数
- 结果 JSON
- 全流程处理日志

这部分主要用于调试和持续优化 AI直播复盘链路。

## 提示词管理

提示词继续统一放在后台 `提示词管理` 菜单中，当前已经拆分为：

- `AI直播复盘 · 高频弹幕筛选`
- `AI直播复盘 · 主分析流程`

这样可以独立调优弹幕筛选和主分析提示词，而不需要改代码重发版。
